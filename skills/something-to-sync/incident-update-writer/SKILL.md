# Incident Update Writer

Use when asked to turn incident notes into a clear stakeholder update.

Collect current impact, affected users or systems, confirmed evidence, suspected cause, mitigation in progress, owner, and next update time.

Write:

1. Status: one sentence with severity and user impact.
2. Evidence: two or three facts, no speculation.
3. Action: what is being done now and by whom.
4. Next update: exact time or trigger.

If impact or owner is unknown, say so plainly and ask for that missing field.
